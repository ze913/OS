Markdown 零基础速查手册（避坑+实用版）
核心前提：Markdown语法符号全是英文半角符号，用中文符号会失效！VSCode内直接写，保存为.md后缀即可预览
一、高频基础语法（必掌握，90%场景够用）
1. 标题（分级清晰，最多六级）
语法：# 加空格 + 标题文字，# 数量对应标题级别
# 一级标题（最大，常用于文档名）
## 二级标题（章节标题）
### 三级标题（小节标题）
#### 四级标题
##### 五级标题
###### 六级标题（最小）
注意事项：# 和文字之间必须加空格，不加会失效；不要越级标题（比如一级后直接三级）
2. 文字样式（重点内容突出）
**加粗文字**  —— 核心重点、关键词
*斜体文字*  —— 备注、次要强调
~~删除线~~  —— 废弃内容、修改标注
==高亮文字==  —— 关键提醒、易错点
注意事项：样式符号紧贴文字，中间无空格；多种样式可叠加，比如加粗斜体
3. 列表（整理内容、分点说明）
无序列表（无序事项、并列内容）
- 列表项1（- 加空格）
- 列表项2
  - 二级子项（缩进2个空格 + -）
  - 二级子项
有序列表（步骤、排序内容）
1. 第一项（数字 + . + 空格）
2. 第二项
3. 第三项
   1. 子项1（缩进2个空格）
   2. 子项2
注意事项：列表符号后必须加空格；子列表只需缩进2/4个空格，无需额外符号
4. 引用（摘抄、备注、说明）
> 一级引用（> 加空格）
> 多行引用直接换行
>> 二级引用（两个>）
注意事项：> 和文字之间加空格，引用内可嵌套其他基础语法
5. 分割线（区分模块）
---  （三个减号，最常用）
***  （三个星号，效果一样）
注意事项：分割线单独占一行，上下空行，排版更整洁
二、代码专属语法（写电梯项目必用）
1. 行内代码（短句代码、函数名）
调用 `dispatch()` 函数实现电梯调度，`Elevator` 是核心类
语法：内容用反引号（`）包裹，反引号在键盘左上角Esc下方
2. 代码块（整段代码、完整类/函数）
```python
# 电梯核心类
class Elevator(QThread):
    def __init__(self, eid):
        self.floor = 1
        self.state = STOP
```
注意事项：
- 代码块用三个反引号包裹，开头反引号后标注语言（python/java/text），实现语法高亮
- 代码块单独占一行，上下空行，避免和其他文字粘连
- 写Python代码时，缩进保持和原代码一致，不破坏格式
三、常用进阶语法（文档整理）
1. 表格（对比、清单类内容）
| 类名 | 核心作用 | 状态 |
| ---- | ---- | ---- |
| Elevator | 电梯运行、任务队列 | 运行中 |
| MainWindow | 界面、调度逻辑 | 就绪 |
注意事项：第二行分隔线可简化，列数对齐；表格内文字居中/左对齐无需额外符号，默认排版即可
2. 任务清单（待办、进度）
- [x] 完成电梯类编写
- [ ] 优化调度算法
- [ ] 测试界面交互
语法：- + 空格 + [空格/字母x] + 空格，[x]代表已完成，[ ]代表未完成
四、高频避坑注意事项（必看，少踩90%的坑）
- 符号全英文：所有#、*、`、-、>都是英文半角，中文符号直接失效
- 空格不能少：标题、列表、引用的语法符号后，必须加空格，否则不识别
- 分段空一行：不同模块、段落之间空一行，避免文字粘连，排版混乱
- 代码块规范：整段代码必须放代码块，不要混在普通文字里，语言标注准确
- 不要随意缩进：普通文字不要随意缩进，只有子列表、代码块需要规范缩进
